================================================================================
                     FUNCTIONAL DIFFERENCES REPORT
                   main.js vs main.py (Our Legacy RPG)
================================================================================

This report compares the functionality of main.js (browser version) with main.py 
(CLI version), focusing on features and capabilities rather than UI text.

================================================================================
1. MISSING FEATURES IN main.js (compared to main.py)
================================================================================

The following features exist in main.py but are NOT implemented in main.js:

1.1 ALCHEMY/CRAFTING SYSTEM
   - main.py: Full alchemy workshop with categories (Potions, Elixirs, 
              Enchantments, Utility)
   - main.js: NOT IMPLEMENTED - No access to crafting system

1.2 DUNGEONS SYSTEM
   - main.py: Complete dungeon system with:
     * Multiple room types (battle, question, chest, trap_chest, multi_choice, 
       empty, boss)
     * Dungeon generation with difficulty scaling
     * Room rewards (gold, items, experience)
     * Dungeon completion tracking
   - main.js: NOT IMPLEMENTED - No dungeon access in menu

1.3 HOUSING/BUILD SYSTEM
   - main.py: Complete housing system with:
     * Housing shop to buy items
     * Build/place items in slots (house, decoration, fencing, garden, 
       farm, training_place)
     * Comfort points calculation
     * View home status
   - main.js: PARTIALLY IMPLEMENTED - buildHome() shows building slots but 
              NO ability to place/remove items, NO housing shop

1.4 TRAINING SYSTEM
   - main.py: Full training ground with:
     * 4 training types (Morning, Calm, Normal, Intense)
     * Dice-based stat improvements
     * Training facility effectiveness based on buildings
   - main.js: NOT IMPLEMENTED - No training menu option

1.5 FARMING SYSTEM
   - main.py: Complete farming with:
     * Plant crops with growth times
     * Harvest crops
     * Sell crops for gold
   - main.js: NOT IMPLEMENTED - No farming menu option

1.6 ELITE MARKET (API MARKET)
   - main.py: Full MarketAPI class with:
     * Multiple API endpoints with fallback
     * 10-minute cooldown
     * Item filtering (type, rarity, class, price)
     * Pagination
   - main.js: MarketAPI imported but:
     * NO visitMarket() method in Game class
     * Market not accessible from main menu

1.7 SETTINGS MENU
   - main.py: Complete settings with mod system toggle and language selection
   - main.js: PARTIALLY IMPLEMENTED - settingsWelcome() exists but:
     * Missing mod system toggle functionality
     * Language menu shows but changes may not persist properly

1.8 MOD MANAGEMENT
   - main.py: Full ModManager with discover, enable/disable mods
   - main.js: ModManager imported but:
     * No discoverMods() implementation visible
     * No mod toggle functionality

1.9 SELL ITEMS AT SHOP
   - main.py: Can sell items from inventory to shops
   - main.js: NOT IMPLEMENTED - No sell functionality

1.10 CLAIM MISSION REWARDS
   - main.py: Separate claim rewards menu
   - main.js: NOT IMPLEMENTED - No claim rewards option

1.11 CUTSCENES SYSTEM
   - main.py: Full cutscene playback with choices
   - main.js: NOT IMPLEMENTED - No cutscene support

1.12 MATERIAL GATHERING
   - main.py: _gather_materials() during exploration
   - main.js: NOT IMPLEMENTED - No material gathering

1.13 QUIT GAME
   - main.py: Proper quit with save prompt
   - main.js: NOT IMPLEMENTED properly - no working quit option

================================================================================
3. GAME MECHANICS DIFFERENCES
================================================================================

3.1 INPUT HANDLING
   - main.py: Synchronous input via input() function
   - main.js: Asynchronous via Promise and custom events

3.2 NAMING CONVENTIONS
   - main.py: snake_case (e.g., current_area, mission_progress)
   - main.js: camelCase (e.g., currentArea, missionProgress)

3.3 PLAYER ATTRIBUTES (Naming differences)
   Python:                  JavaScript:
   - current_area           - currentArea
   - mission_progress       - missionProgress
   - completed_missions     - completedMissions
   - bosses_killed          - bossesKilled
   - housing_owned          - housingOwned
   - building_slots         - buildingSlots
   - comfort_points         - comfortPoints
   - farm_plots             - farmPlots
   - pets_owned             - petsOwned
   - active_pet             - activePet
   - companions_data         - companionsData
   - enemies_data           - enemiesData

3.4 DATA FILE LOADING
   - main.py: Synchronous file reading with json.load()
   - main.js: Async fetch() API with Promise handling

3.5 PRINT/OUTPUT
   - main.py: Direct print() to console
   - main.js: Custom printCallback for DOM manipulation

================================================================================
4. BATTLE SYSTEM DIFFERENCES
================================================================================

The battle systems are very similar but with minor differences:

4.1 BOSS DETECTION
   - main.py: Uses isinstance(enemy, self.game.Boss)
   - main.js: Uses enemy.isBoss property

4.2 DAMAGE CALCULATIONS
   - main.py: Uses int() for Python 3 integer division
   - main.js: Uses Math.floor() for floating-point handling

4.3 ENEMY OBJECT CHECK
   - main.py: Checks hasattr for Boss class
   - main.js: Checks isBoss boolean property

6. SUMMARY
================================================================================

main.js is a PARTIAL PORT of main.py. While the core gameplay loop exists 
(explore, battle, inventory, missions, companions, shops, travel), several 
major features are missing:

FULLY MISSING:
- Alchemy/Crafting
- Dungeons
- Housing placement system
- Training
- Farming
- Elite Market
- Sell items
- Claim rewards
- Cutscenes
- Material gathering

PARTIALLY IMPLEMENTED:
- Housing (can view, cannot modify)
- Settings (limited)
- Mod management (limited)

The JavaScript version appears to be designed for a simplified browser 
experience while main.py is the full-featured CLI version.
